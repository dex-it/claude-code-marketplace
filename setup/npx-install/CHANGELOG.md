# История изменений

## v2.0 - 2025-12-08

### Основные изменения

✨ **Глобальная установка по умолчанию**
- Теперь скрипт устанавливает Context7 с `scope: user` (глобально) по умолчанию
- Context7 будет доступен во всех ваших проектах

✨ **Поддержка параметра --scope**
- Добавлена возможность выбора области установки через параметр командной строки
- Доступные значения: `user`, `local`, `project`

✨ **Улучшенная документация**
- Добавлена справка `--help` в скрипт установки
- Обновлен README.md с примерами использования параметра --scope
- Обновлена шпаргалка cheatsheet.md

### Новые возможности

#### Параметры командной строки

```bash
# Справка
./install-context7.sh --help

# Глобальная установка (по умолчанию)
./install-context7.sh
./install-context7.sh --scope user

# Локальная установка
./install-context7.sh --scope local

# Проектная установка
./install-context7.sh --scope project
```

#### Улучшенная обработка переустановки

- Скрипт теперь определяет текущую область установки
- Предлагает переустановить с новым scope
- Автоматически удаляет старые конфигурации из всех областей

#### Информативный вывод

- Отображает выбранную область установки в начале работы
- Показывает описание области установки
- Выводит подсказки при использовании нестандартных областей

### Миграция с v1.0

Если у вас уже установлен Context7 с локальной областью (v1.0), вы можете:

1. **Переустановить глобально (рекомендуется):**
   ```bash
   ./install-context7.sh
   ```
   Скрипт предложит удалить старую конфигурацию и установить глобально.

2. **Оставить локальную установку:**
   Ничего делать не нужно, Context7 продолжит работать в текущем проекте.

3. **Установить в обеих областях:**
   Это возможно, но не рекомендуется. Локальная конфигурация имеет приоритет над глобальной.
