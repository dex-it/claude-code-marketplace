# dex-autonomous-task-resume

Напоминание о процедуре возобновления `dex-skill-autonomous-task` после компакта контекста.

## Проблема

`dex-skill-autonomous-task` держит процедуру "Возобновление" (перечитать SKILL.md и файл трека,
восстановить состояние из план-файла и тасклиста, сверить ground truth и синхрон трекера) - но
триггер её запуска не технический, а расчёт на то, что модель сама вспомнит выполнить эту
процедуру после того, как контекст уже частично потерян компактом.

## Механизм

Движок на шаге 1 цикла заводит **task-флаг** - `TaskCreate` с `subject: "autonomous-task:
track=<трек>"`. Задачи Claude Code официально переживают компакт контекста ("Tasks persist
across context compactions") и без `CLAUDE_CODE_TASK_LIST_ID` лежат в
`~/.claude/tasks/<session_id>/<id>.json` - директория именована точно `session_id` этой сессии.

Хук на `SessionStart` с `matcher: "compact"` (срабатывает после того, как компакт уже произошёл):

1. Берёт `session_id` из stdin JSON, читает `~/.claude/tasks/<session_id>/*.json`.
2. Ищет незакрытую (`status != completed`) задачу с `subject`, начинающимся на
   `autonomous-task: track=`.
3. Нашёл - инжектит напоминание через `additionalContext` с кодом трека, указывая выполнить
   процедуру "Возобновление".
4. Не нашёл (обычная сессия, либо движок сдал цель и закрыл флаг на шаге 8) - тихо завершается.

Движок закрывает флаг (`TaskUpdate` -> `completed`) на шаге 8 цикла (сдача) - симметрично
заведению на шаге 1.

## Ограничения

- Привязано к task-флагу **этой конкретной сессии** - детерминированно, без разбора "какой из
  нескольких планов относится к этой сессии" (в отличие от файлового подхода по
  `docs/autonomous/plan-*.md`, который не session-specific).
- Формат хранения задач (`~/.claude/tasks/<session_id>/<id>.json`) задокументирован частично
  (официально подтверждено "tasks persist across compactions" и путь `~/.claude/tasks/`, но
  точный per-session layout без `CLAUDE_CODE_TASK_LIST_ID` - по эмпирической проверке, не по
  явной документации) - при смене формата хранения хук может замолчать, деградация тихая.
- Движок обязан закрывать флаг на шаге 8; если цель брошена без сдачи (ручное прерывание), флаг
  останется висеть и будет напоминать в следующей сессии с тем же `session_id` (переиспользование
  `session_id` между разными задачами - редкий кейс, но при `--resume` возможен).
