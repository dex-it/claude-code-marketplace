# Трек «Тест-инжиниринг» - дельта сверх таблицы

> Дополнение к строке «Тест-инжиниринг» таблицы треков в `SKILL.md`. Ядро читает этот файл,
> определив трек (mandatory-Read из шага 1) - до плана. Это **дом тест-дисциплины**: на якорь
> `P-test-matrix` ссылаются `development.md` (добор покрытия) и `mr-review.md` (оценка достаточности).

*Цель = закрыть матрицу покрытия по приоритету, не прогон ради галочки.*

<a id="P-test-matrix"></a>**Матрица покрытия - оси, не одна (happy-path не матрица):**
(1) **happy-path** - основной сценарий; (2) **негатив** - невалидный ввод/ошибки/исключения
дают ожидаемый исход, не happy молча; (3) **краевые** - границы значений, пусто/предел/overflow,
конкурентность если применима; (4) **все ветки логики** - каждая ветвь/провайдер/fallback/retry
прогнана, не только основная. Покрыл подмножество - **скажи, что отрезал** (без тихого усечения),
приоритизируй осознанно. Саморевью: изоляция, не тавтология, антипаттерны.

Это дом тест-дисциплины: «Разработка» и «Ревью MR» опираются на эти оси, не изобретают свои.
