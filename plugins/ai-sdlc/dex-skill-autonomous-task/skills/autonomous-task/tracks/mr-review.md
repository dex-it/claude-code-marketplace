# Трек «Ревью входящего MR» - дельта сверх таблицы

> Дополнение к строке «Ревью входящего MR» таблицы треков в `SKILL.md`. Ядро читает этот файл,
> определив трек (mandatory-Read из шага 1) - до плана. Якорь `P-test-matrix` - в `test-engineering.md`.

*Режим, противоположный разработке: ничего не реализуем, выход = комментарии.* Саморевью =
**фальсифицируй каждую находку** против кода (опровергни перед публикацией - не плоди шум).
Второй круг -> **ре-ревью-агенту дельты**, не первичному.
Бизнес-/intent-неясность по diff -> **тред-вопрос автору** (async), остальное публикуй не
вставая в ожидание. Ревью **закрывается по получении ответов** (треды разрешены) -> двигаешь
задачу. Технику и качество не спрашивают - выносишь находкой.

**Покрытие тестами - отдельный фокус, не только «как написано»:** изменённое/новое поведение
должно иметь тесты через реальный путь; нет/слабое/тавтологичное покрытие, непокрытые краевые
случаи - **находка с severity**, не молчи. Но тесты здесь **не пишем** (read-only) - недостаток
покрытия выносится находкой/задачей трека «Тест-инжиниринг» (он - дом тест-дисциплины, оси
покрытия там, [[P-test-matrix]]).
