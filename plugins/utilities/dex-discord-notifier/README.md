# dex-discord-notifier

Плагин для отправки уведомлений в Discord при работе Claude Code.

## Возможности

### Уведомления (отправка в Discord)

- **Stop** — Claude завершил работу
- **Notification** — Claude ждёт ответа / нужны разрешения
- **SubagentStop** — субагент завершил подзадачу
- **PostCompact** — завершился compact контекста

### Отмена уведомлений (pending-уведомление гасится)

- **PreToolUse** — Claude продолжает работать, уведомление ещё не актуально
- **UserPromptSubmit** — пользователь ввёл сообщение, сессия продолжается

### Содержимое уведомлений

- Последнее сообщение Claude
- Ultrathink (опционально)
- Статус TODO списка
- Используемые инструменты
- Текущий план (если есть)
- Вопросы к пользователю (AskUserQuestion)

## Установка

### 1. Создание вебхука в чате ДС, куда будут приходить сообщения 

1. Создайте свой сервер в дискорд, создайте текстовый канал.
2. Переходим в настройки канала -> Интеграция -> Вебхуки
3. Создаем вебхук, настраиваем имя и ававтарку по желанию
4. Нажимаем "копировать URL вебхука"

### 2. Настройте переменные окружения

```bash
# Добавьте в ~/.bashrc, ~/.zshrc или .env файл проекта
export DISCORD_NOTIFIER_URL="скопированный урл"
```

## Конфигурация

### Обязательные переменные

| Переменная | Описание    |
|------------|-------------|
| `DISCORD_NOTIFIER_URL` | URL вебхука |

### Опциональные переменные

#### События (все включены по умолчанию)

| Переменная | Default | Описание                                |
|------------|---------|-----------------------------------------|
| `DISCORD_NOTIFY_STOP` | `true` | Уведомлять о завершении работы          |
| `DISCORD_NOTIFY_WAITING` | `true` | Уведомлять об ожидании ответа           |
| `DISCORD_NOTIFY_SUBAGENT` | `true` | Уведомлять о субагентах                 |
| `DISCORD_NOTIFY_COMPACT` | `true` | Уведомлять о завершении compact         |

#### Компоненты сообщений

| Переменная | Default | Описание |
|------------|---------|----------|
| `DISCORD_INCLUDE_MESSAGE` | `true` | Включать последнее сообщение |
| `DISCORD_INCLUDE_THINKING` | `false` | Включать Ultrathink |
| `DISCORD_INCLUDE_TODO` | `true` | Включать статус TODO |
| `DISCORD_INCLUDE_TOOLS` | `true` | Включать используемые инструменты |
| `DISCORD_INCLUDE_PLAN` | `true` | Включать план |
| `DISCORD_INCLUDE_QUESTIONS` | `true` | Включать вопросы |

#### Прочие настройки

| Переменная | Default | Описание |
|------------|---------|----------|
| `DISCORD_LANGUAGE` | `ru` | Язык сообщений (`ru` или `en`) |
| `DISCORD_MESSAGE_LIMIT` | `4000` | Лимит символов в сообщении |

## Зависимости

- `curl` - для отправки HTTP запросов
- `jq` - для парсинга JSON

Обычно предустановлены в большинстве Linux/macOS систем.

## Механика доставки

Уведомления не отправляются мгновенно. После каждого события запускается таймер на `DISCORD_NOTIFY_DELAY` секунд (по умолчанию 30). Если за это время придёт новое событие, таймер перезапускается и отправляется только последнее уведомление.

Таймер сбрасывается (уведомление отменяется), если до его истечения:
- Claude начал выполнять следующий tool call (`PreToolUse`)
- Пользователь отправил новое сообщение (`UserPromptSubmit`)

Это означает, что уведомления о промежуточных событиях в длинной сессии не приходят — только финальное.
