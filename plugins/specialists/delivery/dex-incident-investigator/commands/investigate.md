# /investigate

Расследовать инцидент на общем стенде, найти доказанную корневую причину и предложить фикс на источнике. Режим по умолчанию - только чтение; стенд общий.

## Goal

Провести инцидент через фазы агента `incident-investigator`: Access & Map, Problem Specification, Parallel Evidence, Hypotheses & Chain, Falsification, Fix Plan, Report, Gated Fix.

## Input

Аргумент - описание инцидента (симптом, сообщение об ошибке, что и как заметили). Опционально ветка стенда. Стек и связи проектов определяются по манифестам и стенду.

## Output

- Спецификация IS / IS-NOT с моментом начала, хронология, собранные доказательства по источникам
- Причинно-следственная цепочка до структурной причины с severity / confidence
- Проблемы и план фикса на источнике с фальсификацией фикса
- После команды `фиксируй` - серия правок по одной проблеме с верификацией red -> green

## Constraints

- Режим по умолчанию - только чтение; любой эффект на общий стенд - отдельное подтверждение
- Симптом не равно причина: фикс на источнике, не на месте проявления
- Confidence ниже 80 на причине - гипотеза под вопросом, не объявленная причина
- Сбор улик со стенда делегируется инфра-плагинам (kubectl / glab / teamcity / grafana / seq), не дублируется

Команды цикла: `оформляй` (полный отчёт), `фиксируй` (фикс по одной проблеме), `стоп` (прекратить). Делегировать агенту `incident-investigator`.
