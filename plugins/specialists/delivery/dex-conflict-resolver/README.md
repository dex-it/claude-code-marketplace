# dex-conflict-resolver

Языко-агностичное подтягивание базовой ветки в фича-ветку и разрешение конфликтов merge/rebase так, чтобы ни одна функциональность не потерялась ни из базы, ни из фичи. Главный враг не маркеры конфликта, а тихая потеря: автослияние прошло чисто, а сторона молча выпала или сломалась. Чистое автослияние не доказательство сохранности. Заканчивается на локально разрешённой ветке: push (и `--force-with-lease` для rebase) отдельно.

## Команда

`/resolve-conflicts [base-branch] [feature-branch] [merge|rebase]` - интеграция базы и разрешение конфликтов. Режим merge по умолчанию; rebase только для приватной ветки и по политике линейной истории.

## Архитектура

Команда делегирует агенту `conflict-resolver` (Preflight and Safety -> Conflict Preview and Map -> Parallel Conflict Analysis -> Resolution Plan -> Resolve One by One -> Falsification Verification -> Finalize and Loop). Параллельный разбор конфликтов и охота за тихими конфликтами распараллеливаются через `Agent` tool.

Каждый конфликт разбирается отдельно: восстанавливаются намерения base/ours/theirs, оба сводятся в один результат, а не выбирается сторона. modify/delete и rename/rename проверяются на замаскированный переезд - правки переносятся на новый путь, а не выбрасываются. ours/theirs зеркальны в rebase относительно merge.

Верификация доказывает, что не потеряна ни сторона базы, ни сторона фичи: обход обеих дельт хунк за хунком, повтор охоты за тихими конфликтами на результате, `git show --cc` против evil merge, реальный прогон build, тип-чек, линтер, тесты и audit. До команды `делай` рабочее дерево не меняется; финал по `коммить` только при зелёной верификации. Push запрещён: финиш - локально разрешённая ветка. Цикл: `оформляй`, `делай`, `коммить`, `откати`, `стоп`.

## Skills

`dex-skill-git-workflow`, `dex-skill-merge-conflict-resolution` (тихие конфликты, ours/theirs в rebase, modify/delete как переезд, lock-файлы, evil merge), `dex-skill-codebase-conventions` в подготовке; `dex-skill-review-evidence` в разборе и верификации; `dex-skill-no-loose-ends` в разрешении; `dex-skill-output-hygiene` в отчёте.

## Связанные плагины

- `dex-feature-implementer` - реализация фичи до локальных коммитов; конфликт при rebase на свежую базу разрешается этим агентом.
- `dex-self-reviewer` - следующий шаг: pre-push саморевью обновлённой ветки.
