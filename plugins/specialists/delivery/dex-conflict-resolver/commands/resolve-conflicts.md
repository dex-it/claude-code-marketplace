# /resolve-conflicts

Подтянуть актуальную базовую ветку в мою фича-ветку и свести два набора намерений в один результат без потери функционала ни одной стороны. Финиш - локально разрешённая ветка; push отдельно.

## Goal

Провести интеграцию через фазы агента `conflict-resolver`: Preflight and Safety, Conflict Preview and Map, Parallel Conflict Analysis, Resolution Plan, Resolve One by One, Falsification Verification, Finalize and Loop.

## Input

Опционально base-branch (по умолчанию `origin/develop` или `origin/main` по политике проекта), feature-branch (по умолчанию HEAD) и режим merge или rebase (по умолчанию merge). Стек определяется по манифестам проекта.

## Output

- Карта конфликтов по типам и план сведения обоих намерений по каждому
- Список тихих семантических конфликтов без маркеров с file:line
- После `делай` - реальный merge/rebase и разрешение по одному с локальной проверкой
- Раздел верификации: обе стороны целы, сборка и тесты зелёные, evil merge нет
- Финал по `коммить` - merge-коммит или завершённый rebase; push не сделан

## Constraints

- Чистое автослияние не доказывает сохранность; обе дельты проверяются хунк за хунком
- `--ours`/`--theirs` целиком по файлу - только при доказанном полном поглощении
- ours/theirs зеркальны в rebase; не угадывать, читать вывод git
- Confidence ниже 80 в разрешении - вопрос, не компромисс наугад
- Никакого evil merge и push: агент заканчивается на локально разрешённой ветке

Команды цикла: `оформляй` (полный план разрешения), `делай` (реальный merge/rebase по одному конфликту), `коммить` (финализировать при зелёной верификации), `откати` (`--abort` или reset к backup-ref), `стоп`. Следующий шаг - `/self-review`. Делегировать агенту `conflict-resolver`.
