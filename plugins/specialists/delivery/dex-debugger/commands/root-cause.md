# /root-cause

**Goal:** Найти первопричину репортнутого бага по коду и (при запросе) исправить корень с подтверждающим тестом.

**Output:** Pass 1 (Direct Analysis: находки, файл:строка, evidence) → Pass 2 (проверка по skills стека) → Hypothesis (root cause + статус fact-check) → Fix (минимальный diff, red→green тест, статус регрессии). Формат — как в агенте `debugger`.

## Действия

- Воспроизвести: собрать вход (stack trace / лог / шаги / ожидаемое vs фактическое / env), проследить путь до места сбоя; определить стек по манифестам
- Найти первое место в нашем коде в трейсе, собрать первичные находки, проверить по релевантным skills стека
- Свести в гипотезу первопричины «если X, то Y, потому что Z»; утверждение о фреймворке в основе причины — сверить (verified / unverifiable / contradicted)
- При запросе правки — root cause подтверждён (hard gate) → failing-тест через реальный путь (RED) → фикс корня → GREEN → прогон всех тестов

## Notes

- Лечить причину, не симптом; обходной путь — только с явной пометкой
- Не править код без подтверждения пользователя
- Нужны runtime-инструменты (profiler, dump) — сказать и передать профильному диагносту
- Глубину под стек дают профильные skills `dex-skill-<стек>-*` (Pass 2), не отдельный агент на язык; здесь кросс-стековая диагностика
