# /root-cause

**Goal:** Найти первопричину репортнутого бага по коду и (при запросе) исправить корень с подтверждающим тестом.

**Output:** Pass 1 (гипотеза root cause: файл:строка, evidence) → Pass 2 (проверка по skills стека) → Fix (минимальный diff, red→green тест, статус регрессии). Формат — как в агенте `debugger`.

## Действия

- Собрать вход: stack trace / лог / шаги воспроизведения / ожидаемое vs фактическое
- Определить стек по манифестам, найти первое место в нашем коде в трейсе
- Сформулировать гипотезу первопричины, проверить по релевантным skills стека
- При запросе правки — failing-тест через реальный путь (RED) → фикс корня → GREEN → прогон всех тестов

## Notes

- Лечить причину, не симптом; обходной путь — только с явной пометкой
- Не править код без подтверждения пользователя
- Нужны runtime-инструменты (profiler, dump) — сказать и передать профильному диагносту
- Для глубокой .NET-специфики — профильный `bug-hunter`; здесь кросс-стековая диагностика
