# /ef-migration

**Goal:** Создать EF Core миграцию, проверить корректность Up/Down, подготовить к применению.

**Output:** Имя созданной миграции, список изменений схемы (таблицы/колонки/индексы), команды для применения/отката/генерации SQL скрипта.

## Действия

- Найти проект с DbContext
- Создать миграцию: `dotnet ef migrations add <name>`
- Проверить сгенерированный файл: правильные Up(), корректный Down(), нет потери данных
- Сгенерировать идемпотентный SQL скрипт для production

## Notes

- Без аргумента -- спросить имя миграции
- Не применять миграцию автоматически к production
- При конфликтах миграций -- предупредить и предложить решение
