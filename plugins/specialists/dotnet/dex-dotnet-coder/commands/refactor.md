# /refactor

**Goal:** Проанализировать код, найти проблемы качества, предложить и применить рефакторинг.

**Output:** Проблемы по категориям (критично / важно / улучшение) с указанием file:line и предполагаемым исправлением. Итоговая оценка N/10.

## Что проверяется

- Code smells: длинные методы (>50 строк), дублирование, magic numbers
- SOLID violations: SRP, DIP, лишние зависимости
- Performance: async over sync (.Result), N+1, отсутствие AsNoTracking
- Naming: неинформативные имена переменных, нарушение conventions

## Действия

- Без аргумента -- спросить, что анализировать
- Категоризировать: критично / важно / улучшение
- Для каждой проблемы: показать конкретную строку и предложить исправление
- Применить исправления после согласования с пользователем
