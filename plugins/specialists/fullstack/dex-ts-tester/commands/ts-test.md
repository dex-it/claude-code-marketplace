# /ts-test

**Goal:** Запустить тесты в TypeScript/JavaScript проекте, проанализировать результаты и coverage.

**Output:** Сводка: passed/failed/skipped, время, coverage %. Для упавших тестов: имя, ожидание vs результат, предполагаемая причина. Для низкого coverage: файлы ниже 50%.

## Действия

- Определить раннер по package.json / конфигу: Vitest (`vitest.config`) или Jest (`jest.config`)
- Запустить тестовый скрипт проекта (`npm test`) или раннер напрямую (`vitest run --coverage` / `jest --coverage`)
- Для упавших тестов: показать код теста и тестируемого модуля
- Предложить исправление при наличии очевидной причины

## Notes

- Если тесты не проходят type-check -- сначала исправить типы
- При наличии аргумента -- запустить только указанный файл или фильтр (`-t <name>`)
- Coverage считать критичным ниже 50% для модулей с бизнес-логикой
- Флейки (зависят от порядка) -- проверить изоляцию моков/таймеров (см. dex-skill-vitest-jest)
