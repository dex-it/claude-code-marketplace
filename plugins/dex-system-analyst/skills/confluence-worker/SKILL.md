# Confluence Worker — руководство (рус.)

Краткое описание
- Этот скилл предназначен для агентов, которые ищут, создают и редактируют контент в Confluence. Фокус — аккуратная структура документов, переиспользуемые шаблоны, контроль ссылок и совместная работа.

Кому полезно
- Автоагентам, техническим писателям, аналитикам требований и тимлидам, которым нужно поддерживать документацию в актуальном и читаемом виде.

Основные принципы работы
- Всегда сначала искать существующий материал — избегать дублирования.
- Предлагать слияние/обновление существующих страниц, а не создавать большие монолитные документы.
- Подтверждать значительные изменения с заказчиком/автором контента.

## Кросс‑ссылки между документами
- При обнаружении упоминания ключевой сущности/алгоритма автоматически искать существующую страницу (по title, aliases, labels, page properties) и предлагать ссылку.
- Использовать устойчивые internal‑links на основе pageId/перманентного URL (не только по заголовку) чтобы ссылки оставались валидными при переименовании.
- Предлагать варианты при множественных совпадениях: спросить пользователя, что выбрать.
- Обновлять и валидировать внутренние ссылки при перемещении/слиянии страниц; логировать изменения ссылок (action, fromPageId, toPageId, status).
- Логика предлагаемых ссылок должна учитывать релевантность (контекст, теги, авторство) и показывать метрику совпадения пользователю.

Триггеры / команды
- `confluence.search(query, space)` — поиск страниц по `query` в пространстве `space`.
- `confluence.create(page)` — создать страницу, где `page` содержит `title`, `parentId`, `content` (wiki/markdown) и `labels`.
- `confluence.update(pageId, patch)` — частичное обновление содержимого или метаданных страницы.
- `confluence.append(pageId, content)` — добавить раздел к существующей странице (с явной ссылкой на источник).
- `confluence.move(pageId, newParentId)` — переместить страницу в другую ветку структуры.

Формат входа/параметров
- `title` (string, обязательный): заголовок страницы.
- `parentId` (string, необязательный): id родительской страницы; если отсутствует — создать в корне пространства.
- `content` (string, обязательный): основной HTML/Confluence Storage Format или markdown (в зависимости от интеграции).
- `labels` (array[string], необязательный): набор меток для поиска и фильтрации.
- `visibility` (enum, опционально): `public`/`restricted` — настройка видимости при поддержке системой.

Права доступа и безопасность
- Требуются credentials с правами на чтение и запись в целевом пространстве Confluence.
- Никогда не логируйте секретные токены или содержимое, помеченное как конфиденциальное.
- Перед массовым изменением страниц запрашивайте подтверждение пользователя.

Стандарты форматирования
- Используйте согласованную иерархию заголовков: `h1` → `h2` → `h3`.
- Для технических фрагментов используйте блоки кода.
- Таблицы — для структурированных данных (параметры API, поля формы и т.п.).
- Добавляйте блок «Последнее обновление» и авторство внизу страницы.
- Используйте `page properties` и метки для фильтрации и обнаружения.

Поведение при дублировании контента
- При обнаружении похожих страниц предложите варианты:
  - Обновить существующую страницу (предпочтительно).
  - Создать дочернюю страницу и пометить ссылкой на исходную.
  - Создать новую страницу только с явным указанием причины (например, другое пространство или аудитория).

Примеры (JSON запросы к API / псевдокод)
- Поиск страниц:
```json
{ "action": "confluence.search", "query": "API integration guide", "space": "DEX" }
```
- Создание страницы:
```json
{
  "action": "confluence.create",
  "page": {
    "title": "Интеграция с API",
    "parentId": "4567",
    "content": "# Интеграция\n\nОписание...",
    "labels": ["api","integration"]
  }
}
```
- Частичное обновление (патч):
```json
{ "action": "confluence.update", "pageId": "1234", "patch": { "append": "\n## Примечания\nДополнительно..." } }
```

Тестирование и валидация
- Перед публикацией большого изменения сохраняйте черновик и показывайте diff пользователю.
- Проверяйте, что внутренние ссылки (relative links) корректны и не ведут на удалённые/удалённые страницы.
- Автотесты: мок Confluence API + тесты на idempotency для операций `create`/`update`.

Логирование и мониторинг
- Логировать: action, pageId (если есть), статус ответа API, время выполнения.
- Не логировать: токены, приватные секции контента.

Ограничения и ошибки
- Обрабатывать коды 4xx/5xx от Confluence API: ретраи для 5xx с экспоненциальной задержкой.
- Валидация входа: максимальная длина заголовка/контента, допустимые метки.

Рекомендации по структуре страницы (шаблон)
```markdown
# Заголовок страницы

Краткое описание (1–2 предложения)

## Контекст
- Зачем нужна эта страница
- Владелец / контакт

## Содержание
- Подробные инструкции или описание

## Примеры
- Примеры конфигурации, вызовов API

## История изменений
- Дата — Краткое описание изменений — Автор
```
