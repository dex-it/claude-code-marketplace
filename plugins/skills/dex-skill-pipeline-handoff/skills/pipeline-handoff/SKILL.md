# Контракт стыка между агентами-узлами (трек «Разработка»)

Узел конвейера = агент с сигнатурой (вход + выход). Этот skill - **общий словарь
полей и правило стыка**, чтобы выход узла N принимался входом узла N+1 без потери
данных. Сигнатуру **каждого** узла (что он берёт/даёт) держит сам агент: сжато в
`description` (оркестратор видит до спавна), полно в фазах тела. Здесь - что поля
**значат** и как реагировать на неполный вход. Маршрут (кто за кем) решает
оркестратор, не узел.

> Дом маршрутизации и сборки цепочки - оркестратор (внешний `autonomous-task`).
> Здесь только форма стыка. «Следующий узел» узлом не задаётся.

## Правило стыка (для всех рёбер)

1. **Выход = именованные обязательные поля** ребра. Отсутствующее поле -> явный
   статус (`n/a` + причина / `unverified`), не молчание.
2. **Формат свободный, состав обязателен.** Излагать как удобно; перечень полей -
   нет.
3. **Приёмник проверяет вход** - сверяет пришедшее с обязательными полями своего
   входа. Недостающее обязательное поле НЕ домысливается.
4. **`mode: interactive | autonomous`** в каждом прямом handoff задаёт реакцию на
   нехватку. Источник проставляет; дефолт - `interactive` (лишний halt дешевле
   молчаливого допущения). Узел режим не угадывает - читает из handoff.

## Реакция приёмника на нехватку обязательного поля

| Класс поля | `interactive` | `autonomous` |
|---|---|---|
| `[blocking]` (требования, success criteria) | halt + вернуть запрос наверх | явное допущение + громкая пометка, не halt |
| `[default-ok]` (non-goals, ADR, остатки) | дефолт + пометка | дефолт + пометка |

**«Вернуть наверх != спросить юзера».** Узел-субагент канала к юзеру не имеет.
В `interactive` он возвращает запрос **источнику вызова** (оркестратору/главному
циклу), тот решает - спросить юзера или дополнить сам. В `autonomous` эскалировать
некому: фиксируй допущение явно и продолжай, иначе цепочка замрёт.

## Словарь полей

Поле сверяется **по смыслу, не по имени**: источник может назвать иначе - засчитывать
синоним, если несёт тот же смысл.

| Поле | Смысл | Синонимы источника |
|---|---|---|
| `requirements R/I` | что реализовать: явные (R) + неявные из проекта (I) | scope инкремента + Deep Dive (architect); ТЗ |
| `success criteria` | проверяемый, измеримый критерий «готово» | DoD; Success metric; acceptance criteria; для bug-fixer - исходные баги red->green |
| `non-goals` | что НЕ делать | scope «что не входит» |
| `key decisions` | принятые решения + релевантные `Accepted` ADR | Deep Dive (storage/API/cache/sharding) |
| `constraints/risks` | ограничения, риски | Risks инкремента |
| `diff-scope` | изменённые файлы + ветка/база | - (self-reviewer берёт сам из git, ему не передаётся) |
| `intent` | источник намерения для intent-gate (план/задача) | success criteria; ссылка на ТЗ |
| `run-status` | результат build/test/lint | зелёный/красный + что |
| `anchor` | привязка находки | `file:line` |
| `closure` | когда находка считается закрытой | критерий исправления |
| `severity` | критичность | CRITICAL/HIGH/MEDIUM/LOW |

## Каталог рёбер (типовые стыки, не жёсткий маршрут)

### План -> Код (architect/оркестратор -> coder)
- `[blocking]` `requirements R/I`; `[blocking]` `success criteria`;
  `[default-ok]` `non-goals`, `key decisions`, `constraints/risks`.

### Код -> Тест (coder -> tester)
- `[blocking]` `diff-scope` (что тестировать); `[blocking]` публичные контракты;
  `[blocking]` `success criteria` как **оракул тестов** (критерии приёмки, НЕ план
  реализации); `[default-ok]` что кодер не покрыл.

### -> Ревью (coder/bug-fixer -> self-reviewer)
- `[blocking]` `intent` (для intent-gate); `[default-ok]` `run-status`, остатки.
- **diff НЕ передаётся** - self-reviewer берёт сам из git (committed+staged+worktree).
  Транспорт через git, не текст. Нужно только намерение, иначе intent-gate не с чем
  сверять.

### Возврат Ревью/Тест -> Код
- Каждая находка: `anchor` + `closure` + `severity`. Без `anchor` или `closure`
  находка НЕ передаётся (закрывает «голую находку»).

### Возврат Код -> План
- что невыполнимо/противоречиво (конкретный пункт плана); почему (контракт/стек/
  недостающее решение); чего не хватает. Кодер не домысливает план - возвращает.

## Mid-flight: вопрос возник в ходе работы

Тот же mode-aware механизм: `interactive` + блокер -> halt + вернуть наверх;
`autonomous` -> явное допущение + громкая пометка. Не-блокер -> дефолт + пометка
в обоих режимах. Возврат наверх != вопрос юзеру.
