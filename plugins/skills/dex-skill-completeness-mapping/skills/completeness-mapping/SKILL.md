# Полнота карты — как найти ВСЕХ потребителей данных и управления

Перед тем как сказать «всех читателей и писателей нашёл», «все вызывающие учтены» или
«охват полный» при сквозной правке **любой кодовой структуры** — поля, аргумента,
сигнатуры функции/метода, типа, enum, константы, события, эндпоинта, конфиг-ключа,
колонки БД — обязательный **completeness-gate**. Skill для того, кто строит карту изменений: разработчик,
системный аналитик, автор плана или спеки.

**Главный принцип:** перечень потребителей выводится **из самой структуры** (кто
читает/пишет данные, кто вызывает функцию, кто конструирует/сопоставляет тип), а **не**
из формулировки задачи — задача
называет очевидное, полный список знает только код. Список из разведки/тикета —
гипотеза, не факт; доказывать чтением кода обеих сторон контракта.

**Почему лестница, а не один поиск:** ни один способ поиска не полон. Authoritative-слой
(type-resolved навигация языкового сервера) поднимает пол и даёт каркас без шума;
эвристические слои добирают то, к чему он слеп — но сами шумят или видят только форму.
Слепая зона у каждого слоя своя, поэтому именно связка, сверху вниз.

## Ось 1 — Data flow: кто читает и пишет данные

Лестница из четырёх слоёв. Шов между слоями — строка проекции (`Dst = src.Field`):
она попадает в наборы ссылок обоих концов и сшивает цепочку.

| Слой | Чем искать (роль) | Ловит | Слеп к |
|------|-------------------|-------|--------|
| 1. Authoritative symbol-layer | type-resolved references языкового сервера (LSP findReferences), НЕ текст/форма | прямые символьные read/write, без шума | переход через `DTO = entity.Prop` (per-symbol), by-convention, wire |
| 2. Форма обращения | структурный AST-поиск (паттерн вида `$X.Field`), не текст | mapping-лямбды, форму доступа | значение вне формы (локальная, позиционный аргумент) |
| 3. Текст по значению | regex по тексту — код И не-код | строки, миграции, raw SQL, wire-имена на границе, dead-код | однофамильцы (шум), символ не различает |
| 4. Общий проектор | reference-поиск + структурный обзор (codemap/импорты) по общим мапперам | один маппер → N эндпоинтов | то, что не через общий маппер |

**Граница слоя 1 — per-symbol.** Authoritative-references НЕ пересекает присваивание
`DTO = entity.Prop`: потребитель через DTO сидит на *другом* символе. Полное покрытие =
идти **по цепочке хопами** (entity → строка-проекция → DTO → контроллер → wire), на
каждом хопе свой reference-поиск.

## Ось 2 — Control flow: кто вызывает, все ветки, эффекты

Симметричная лестница для управления — её пропускают чаще всего, считая «данные нашёл,
значит всё».

| Слой | Чем искать (роль) | Ловит | Слеп к |
|------|-------------------|-------|--------|
| 1. Authoritative call-hierarchy | incoming calls языкового сервера (LSP incomingCalls), НЕ эвристический call-graph | прямые вызовы, type-resolved | рефлексия, DI-резолв |
| 2. Форма вызова | структурный AST-поиск по сигнатуре/имени | перегрузки, вызовы через делегат | вызов через индирекцию/интерфейс |
| 3. Текст по имени | regex по тексту (имя метода/роута, конфиги) | строковые роуты, конфиг-driven хендлеры, fire-and-forget, подписки на события | шум однофамильцев |
| 4. Ветки и эффекты внутри | CFG/срез внутри функции (control-flow-инструмент) | непокрытые ветки, ранние return, побочные эффекты на каждом пути | межпроцедурное — добирает слой 1 |

**Authoritative ≠ эвристический call-graph.** Слой 1 — call-hierarchy языкового
сервера (type-resolved). Эвристический анализатор call-graph (по форме/индексу, не через
компилятор) НЕ authoritative: на ряде стеков он не видит статические helper-методы и
выдаёт неполный, но непустой результат — соблазн принять его за слой 1 и закрыть гейт.
Имя инструмента вида «impact / кто вызывает» не делает его authoritative. Статики,
вызовы через рефлексию и DI authoritative-слой тоже добирает не везде — для них слои 2–3
обязательны, а не вывод «вызывающих нет».

## Completeness-gate: запрет тихого пропуска

Гейт закрывает лазейку «прогнал один поиск → объявил готово».

- **Authoritative-слой обязателен и реализуется языковым сервером (LSP), не эвристикой.**
  Недоступен (нет языкового сервера для стека) — это не «пропустить шаг», а зафиксировать
  статус `authoritative-unavailable`, переключиться на усиленный добор слоями 2–4 и
  пометить покрытие как **degraded**. Подмена слоя 1 эвристическим анализатором (call-graph
  по форме/индексу) — это тоже `authoritative-unavailable`, а не закрытый слой 1: гейт не
  считается пройденным. Тихо продолжить на одной эвристике — запрещено.
- **Сначала раздобудь authoritative-инструмент, потом решай о доступности.** Языковой
  сервер часто не подключён в сессии сразу (deferred-тул — вытащить по реальному имени тула (не по строке «LSP»), не по
  ключевым словам «кто вызывает»: иначе всплывут громкие эвристические тулы). «Не вижу
  сразу» ≠ `authoritative-unavailable` — сперва попытка получить.
- **Нет инструмента слоя → слой degraded, не пропуск.** Деградация не только у слоя 1:
  слой 2 без структурного AST-поиска, слой 4 без control-flow-инструмента — тоже degraded,
  а не «прогнан». Сквозь все слои полезны ещё два класса: **семантический поиск по коду**
  (когда точное имя символа неизвестно — переименован, ищешь «где считается сумма») и
  **поиск по не-кодовым артефактам** (миграции, raw SQL, конфиги, OpenAPI/proto, wire-схемы
  — потребитель часто там, а не в исходниках).
- **Degraded — не молчаливая норма: сообщи и верни решение.** Если нужный инструмент
  (языковой сервер слоя 1 или любой класс выше) раздобыть не удалось, явно сообщи
  вызывающему (человеку или оркестратору): какой слой degraded и какого класса инструмента
  не хватает; для полного охвата его надо установить/подключить. Назови класс инструмента,
  не молчи об ограничении. Решение — ставить и перепройти или принять degraded — за
  вызывающим; устанавливать инструмент самовольно (это меняет систему) нельзя — предложить
  и дождаться подтверждения.
- **Каждый слой каждой оси — прогнан или помечен явным статусом** (`n/a` + почему,
  `unverifiable`). Молчание о слое = слой не закрыт.
- **«Готово» — только когда обе оси закрыты.** Нашёл всех читателей поля, но не проверил,
  все ли ветки его обработки задеты, — охват не полный.
- **Перечень — из объекта, не из задачи.** Если список потребителей совпал с тем, что
  назвал тикет, и ни один слой не добавил нового, — это сигнал недопроверки, а не успеха.

## Смежные оси планирования: reach — не всё

Этот skill закрывает **reach** (кого задену). Планирование правки опирается ещё на оси
ниже — здесь они как чеклист с указателем дома, механику каждой держит профильный skill,
а не этот. Ось без дома помечена явно: её добираешь сам, не считая «нет skill = не надо».

| Ось | Вопрос планирования | Где механика |
|-----|---------------------|--------------|
| Origin / провенанс | откуда значение приходит до моей точки, каким инвариантам на входе можно верить | этот skill, слой «писатели» data flow — но как вопрос доверия инварианту проверяй отдельно |
| Контракт / границы | меняется ли наблюдаемое поведение на границе (API, wire, схема БД, событие) | см. dex-skill-contract-drift |
| Инварианты / предусловия | не ломаю ли допущение потребителей (nullability, порядок, диапазон, идемпотентность) | частично dex-skill-contract-drift (nullability на границе); **языко-нейтрального дома нет — проверяй сам** |
| Состояние / миграция | что с уже существующими данными в БД, кэше, очереди, сериализованных снапшотах при смене формата/семантики поля | схема БД — см. dex-skill-dotnet-ef-core (.NET); данные в кэше/очереди/снапшотах — **дома нет, проверяй сам** |
| Конкурентность / порядок | ломает ли правка предположения о потокобезопасности, атомарности, порядке, гонках | см. dex-skill-dotnet-async-patterns (.NET); **языко-нейтрального дома нет** |
| Отказ / откат | как откатить, что при наполовину применённой правке, совместимость старого+нового кода во время деплоя | см. dex-skill-distributed-resilience |
| Хвосты | заглушка, silent fallback, недоделка от самой правки | см. dex-skill-no-loose-ends |

Дыра «дома нет» — не разрешение пропустить ось, а сигнал: добор ручной, результат
фиксируется явным статусом, как и в gate выше.
