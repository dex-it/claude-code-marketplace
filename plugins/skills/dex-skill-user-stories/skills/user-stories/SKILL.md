# User Stories — ловушки

## Правила

- Format: As a [role] I want [goal] So that [value]
- AC обязательно: Given-When-Then для каждого сценария
- Покрывай: happy path + alternative + error + edge case
- Технологии — в Technical Notes, не в story description
- 1-8 SP, иначе split

## INVEST нарушения

| Критерий | Нарушение | Как исправить |
|----------|-----------|---------------|
| Independent | "Зависит от story X" | Включи зависимость в scope или объедини |
| Negotiable | Детальная спецификация на 3 страницы | Story = обещание разговора, детали при refinement |
| Valuable | "Рефакторинг auth-модуля" | "Пользователь может войти за <3 сек" |
| Estimable | "Слишком много неизвестных" | Spike story для исследования, потом оценка |
| Small | 13+ SP | Split по workflow/rules/data |
| Testable | "Работает корректно" | Given-When-Then с конкретными значениями |

## Частые ошибки написания

```
Плохо: "Как пользователь, я хочу страницу настроек"
// Нет value — ЗАЧЕМ нужна страница?
Хорошо: "Как пользователь, я хочу настроить уведомления,
         чтобы получать только релевантные оповещения"

Плохо: "Как разработчик, я хочу внедрить Repository pattern"
// Техзадача, пользователь не причём
Хорошо: → Это Task внутри story, не отдельная story

Плохо: AC: "Всё работает правильно"
// Не тестируемо
Хорошо: AC:
  Given я на странице логина
  When ввожу неверный пароль 3 раза
  Then аккаунт блокируется на 15 минут
  And вижу сообщение "Аккаунт заблокирован. Попробуйте через 15 минут"

Плохо: AC покрывает только happy path
// Нет error/edge cases
Хорошо: AC включает:
  - Happy path: успешный логин
  - Error: неверный пароль
  - Edge: блокировка после N попыток
  - Edge: истёкший токен
  - Security: SQL injection в поле email
```

## AC — пропущенные сценарии

| Часто забывают | Пример |
|---------------|--------|
| Пустые данные | Список без элементов — [] или "Нет данных"? |
| Concurrent access | Два пользователя редактируют одновременно |
| Длинный текст | Имя 500 символов — обрезать или ошибка? |
| Спецсимволы | Имя с апострофом: O'Brien |
| Часовые пояса | Дата создания — в каком timezone? |
| Пагинация граница | Ровно pageSize элементов — есть "Next" или нет? |
| Отмена действия | Можно ли отменить? Undo? |

## Чек-лист

- [ ] Story даёт value пользователю (не техзадача)
- [ ] INVEST: все 6 критериев проверены
- [ ] AC: happy path + error + edge cases
- [ ] AC: конкретные значения (не "правильно работает")
- [ ] Нет технологий в description (Redis, RabbitMQ → Technical Notes)
- [ ] Size: 1-8 SP (13+ → split)
- [ ] Пропущенные сценарии проверены (пустые данные, concurrent, спецсимволы)
